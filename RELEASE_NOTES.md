# 文件迅传 v1.1.0 发布说明

**发布日期**：2026年6月17日  
**版本类型**：功能增强版本

---

## 版本概览

v1.1.0 是一个重要的功能增强版本，在保持原有简洁高效特性的基础上，新增了IP追踪、下载统计等实用功能，并完善了中文文件名编码支持。

---

## 新增功能

### 1. IP追踪系统
- **自动记录上传IP**：每次文件上传时自动记录上传者IP地址
- **代理环境支持**：兼容 X-Forwarded-For、X-Real-IP 等代理头
- **后台显示**：管理页面可查看每个文件的上传IP地址
- **应用场景**：
  - 追溯文件来源
  - 安全审计
  - 异常行为检测

### 2. 下载统计增强
- **实时次数统计**：精确记录每个文件的下载次数
- **时间追踪**：记录最后下载时间
- **相对时间显示**：智能显示"5分钟前"、"2小时前"等友好格式
- **总览面板**：后台首页显示总下载次数统计

### 3. 后台管理升级
- **新增数据列**：
  - 上传IP地址
  - 下载次数（绿色加粗突出显示）
  - 最后下载时间（相对时间格式）
- **统计卡片扩展**：新增"总下载次数"统计项
- **响应式布局**：统计卡片支持自适应换行

---

## 优化改进

### 1. 中文文件名编码完善
- **智能识别**：GBK/UTF-8 编码自动检测和转换
- **多场景支持**：
  - UTF-8字节被当作Latin-1传输
  - UTF-8字节被当作GBK解码
  - 直接GBK编码的文件名
- **评分机制**：优先选择包含常用汉字的解码结果
- **零配置**：无需手动设置，自动处理

### 2. 文件大小显示修复
- **异步写入等待**：确保文件完全写入后再获取大小
- **轮询机制**：50ms间隔检查文件写入状态
- **超时保护**：10秒超时机制防止无限等待
- **日志增强**：详细记录文件大小和写入状态

### 3. 数据库结构升级
- **新增字段**：`upload_ip TEXT`
- **自动迁移**：首次启动时自动添加新字段
- **向后兼容**：保留所有历史数据

## 技术细节

### API变更

#### 上传接口 (`POST /upload`)
```json
// 数据库记录新增字段
{
  "upload_ip": "192.168.1.100"
}
```

#### 文件列表接口 (`GET /api/files`)
```json
[
  {
    "id": "文件ID",
    "originalName": "原始文件名",
    "size": 1048576,
    "uploadTime": "2026-06-17T12:00:00.000Z",
    "downloadCount": 5,
    "lastDownload": "2026-06-17T14:30:00.000Z",
    "uploadIP": "192.168.1.100",
    "downloadUrl": "/d/abc123"
  }
]
```

### 数据库Schema
```sql
CREATE TABLE files (
    id TEXT PRIMARY KEY,
    short_id TEXT UNIQUE,
    original_name TEXT NOT NULL,
    stored_name TEXT NOT NULL,
    size INTEGER NOT NULL,
    upload_time TEXT NOT NULL,
    mime_type TEXT,
    download_count INTEGER DEFAULT 0,
    last_download TEXT,
    upload_ip TEXT                    -- 新增字段
);
```

## 升级指南

### 从 v1.0.x 升级

1. **备份数据**
   ```bash
   cp files.db files.db.bak
   ```

2. **更新代码**
   ```bash
   # 替换 server.js 和 public/admin.html
   ```

3. **安装依赖**（如有新增）
   ```bash
   npm install --production
   ```

4. **启动服务**
   ```bash
   npm start
   ```

5. **自动迁移**
   - 首次启动时会自动添加 `upload_ip` 字段
   - 历史文件的 upload_ip 显示为"未知"

### 注意事项

- 数据库迁移是自动的，无需手动操作
- 升级前的文件不会有上传IP记录
- 建议在生产环境部署前进行测试
